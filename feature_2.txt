[Stage 2]
Project Context:
We are continuing the development of "ContigNexus" to Stage 2. In Stage 1, we established a NetworkX-based Python script that maps physical (GFA) and biological (CRISPR-Phage) connections. Now, we will add the "Statistical & Metabolic Engine". This engine is designed for multi-sample datasets to find contigs that form "Metabolic Guilds" (ecological working groups) based on cross-sample co-abundance correlation and functional complementarity.

Main Objective of Stage 2:
To build a high-performance Python module using pandas and scipy to calculate contig co-abundance correlations, verify metabolic pathway compatibility, and inject these new connections into the NetworkX object inherited from Stage 1.

Additional Required Input Files:

coverage_table.tsv: An abundance matrix table (Column 1: Contig_ID, subsequent columns: Depth/Coverage across Sample_1, Sample_2, etc. Standard output from tools like jgi_summarize_bam_contig_depths).

functional_annotations.tsv: A contig functional annotation table (Column 1: Contig_ID, Column 2: Comma-separated list of KEGG KO / MetaCyc terms).
