[Stage 1]
Project Context:
I am building a Python-based bioinformatics tool called "ContigNexus" (Stage 1). This tool aims to reconstruct inter-contig interactions from low-depth metagenomic data (without the MAG binning process). In Stage 1, the tool will combine two Evidence Layers into a single interactive graph:

Physical Evidence: Parsing the Assembly Graph (.gfa) file to find physically connected contigs.

Biological Evidence (CRISPR-Phage): Finding CRISPR spacers in bacterial contigs and mapping them to viral contigs.

Main Objective of Stage 1:
To create a Python pipeline (using NetworkX) that takes raw assembly files as input, processes interaction detection, and outputs an Edge List table along with an interactive graph visualization (HTML).

Required Input Files:

assembly_graph.gfa: The graph file from the assembler (e.g., MEGAHIT/metaSPAdes).

contigs.fasta: All assembled contigs.

viral_contigs.fasta: A subset of contigs predicted as viral (e.g., from geNomad output).

Algorithm & Logic Instructions (Step-by-Step):

Module 1: GFA Parser (Physical Evidence)

Use Python to parse the .gfa file.

Extract S (Segment/Node) lines to store contig lengths.

Extract L (Link/Edge) lines to find overlap connections between contigs.

Inject this data into a NetworkX object as Edges with the attribute type="physical_overlap".

Module 2: CRISPR-Phage Miner (Biological Evidence)

Create a lightweight Python wrapper to run minced (or a similar CRISPR detection tool) against contigs.fasta.

Extract the spacer DNA sequences from the minced output and save them to a spacers.fasta file.

Create a Python wrapper to run blastn (using custom parameters for short sequences: -task blastn-short -evalue 1e-3) to query spacers.fasta against viral_contigs.fasta.

Filter the BLAST results (keep only alignments with identity > 95% and near-full coverage).

Add the filtered results to the same NetworkX object as Edges with the attribute type="crispr_targeting".

Module 3: Graph Integration & Export

Merge nodes and edges from Module 1 and Module 2.

Export the graph into two output formats.

Output Specifications:

TSV Table (contignexus_edges.tsv):

Columns: Source_Contig, Target_Contig, Evidence_Type (Physical / CRISPR), Weight_or_Identity, Source_Length, Target_Length.

Interactive Visualization (contignexus_network.html):

Use the PyVis library (based on vis.js) to render the NetworkX object into an interactive HTML file.

Visual Rules:

Node Shape: Circle for Bacterial/Unknown Contigs, Triangle for Viral Contigs.

Node Size: Proportional to contig length (log scale).

Edge Color & Style: * Solid thick gray line (#CCCCCC) for physical_overlap.

Dashed red line (#FF0000) with a directed arrow from Bacteria → Virus for crispr_targeting.
